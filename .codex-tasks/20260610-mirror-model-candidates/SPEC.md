# Mirror Model Candidates

## Goal

镜像分组配置界面不要求管理员纯手写模型名。保留手动输入，同时按源分组/目标平台自动加载可选的对外模型，让管理员可以从已有账号模型映射或分组自定义 /v1/models 候选中选择。

## Requirements

- 镜像分组弹窗继续支持手动输入模型映射。
- 根据原分组与目标平台自动加载对应平台的可选模型。
- 优先复用现有账号 `model_mapping` / 分组 models-list 候选能力，不重复造数据源。
- 管理员能把“客户端对外模型”从候选列表中选择出来。
- 不能破坏现有镜像创建/关闭接口。
- 前端验证覆盖关键逻辑。
